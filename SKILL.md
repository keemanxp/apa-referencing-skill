# APA 7th Edition Referencing and Citation Skill

**Author:** Chuah Kee Man | **Based on:** APA Publication Manual, 7th Edition

This skill enables Claude to format, check, correct, and verify references and citations following APA 7th Edition rules. It serves anyone doing academic writing — students, lecturers, researchers, or professionals.

## Core Capabilities

1. **Format references** — Generate correctly formatted reference list entries and in-text citations from source details the user provides.
2. **Check and correct** — Review existing references or reference lists, identify APA formatting errors, and produce corrected versions with explanations.
3. **Verify authenticity** — Use web search to confirm that each reference actually exists. Flag fabricated or unverifiable references and suggest close matches where possible.
4. **Convert styles** — Convert references from other citation styles (e.g. MLA, Chicago, Harvard) into APA 7th Edition format.

## How to Use This Skill

### Step 1: Identify the Task

Determine what the user needs:

- **"Format this"** — User gives source details (author, title, year, etc.) and wants a properly formatted reference and/or in-text citation.
- **"Check this"** — User pastes one or more references and wants them reviewed for APA compliance.
- **"Is this real?"** — User wants verification that references are not fabricated.
- **"Convert this"** — User has references in another style and wants APA 7th Edition format.
- **"Cite this URL/DOI"** — User gives a link and wants a full APA reference generated. Use web_fetch to retrieve metadata first.

### Step 2: Identify the Source Type

Determine which source type the reference belongs to. The full formatting rules for each type are in:

The supported source types are:

### Step 3: Apply Formatting Rules

Read `references/apa7-formatting-rules.md` for the exact formatting template for the identified source type. Apply the rules precisely — APA formatting is strict and details matter.

### Step 4: Verify References (Critical Step)

This step is essential whenever the user provides references that may have been generated by AI or are otherwise suspect. It is also good practice for any reference-checking task.

**Verification process:**

1. **Search for the reference** using web_search. Construct a query from the author surname(s), year, and key title words. For example: `Washington 2014 cyberbully higher education Adult Learning`
2. **Check DOIs** — If a DOI is provided, fetch `https://doi.org/[DOI]` to confirm it resolves to the correct article.
3. **Cross-check details** — Verify that the author names, year, title, journal/publisher, volume, issue, and page numbers all match.
4. **Flag discrepancies** — If any detail doesn't match (e.g. wrong year, slightly different title, non-existent journal), flag it clearly.

**When a reference cannot be verified:**

- State clearly: "⚠️ This reference could not be verified."
- If a close match is found (e.g. similar title, same author but different year), present the close match and ask the user to confirm.
- If no match is found at all, warn the user that this reference may be fabricated and suggest they double-check the original source.
- Never silently pass an unverifiable reference as correct.

**When generating references from scratch (e.g. Claude is asked to "find me a source about X"):**

- Always use web_search to find a real source first.
- Never invent or fabricate any part of a reference — not the authors, not the title, not the DOI, not the page numbers.
- If you cannot find a suitable real source, say so. Do not make one up.

## General APA 7th Edition Rules

These rules apply across all source types. For type-specific formatting, always consult `references/apa7-formatting-rules.md`.

### Author Names
- Use surname followed by initials: `Smith, J. A.`
- Separate multiple authors with commas; use `&` before the last author.
- For 1–2 authors: list all authors in every in-text citation.
- For 3+ authors: use `et al.` after the first author in in-text citations (e.g. `Ewert et al., 2014`).
- In the reference list, always list all authors (up to 20). For 21+ authors, list the first 19, then an ellipsis, then the last author.

### Titles
- **Books, reports, theses, webpages:** Italicise the title. Use sentence case (capitalise only the first word, first word after a colon, and proper nouns).
- **Journal articles, chapters:** Do not italicise the article/chapter title. Use sentence case.
- **Journal names:** Italicise the journal name. Use title case (capitalise major words).
- **Journal volume:** Italicise the volume number. Do not italicise the issue number (in parentheses).

### DOIs and URLs
- Always present DOIs in the format: `https://doi.org/10.xxxx/xxxx`
- Do not put a full stop after a DOI or URL.
- If a URL is very long, a URL shortener (e.g. bit.ly) may be used, but the link must be accessible and point directly to the source.
- Do not include database names or URLs for works retrieved from academic databases (unless it is an open access article without a DOI).

### Dates
- Year only for most sources: `(2019)`.
- Full date for newspapers, magazines, blog posts, social media: `(2019, March 22)`.
- Use `(n.d.)` when no date is available, and include a retrieval date.

### Publisher
- In APA 7th Edition, the place of publication is no longer required. Only include the publisher's name.

### Hanging Indent
- Reference list entries use a hanging indent (first line flush left, subsequent lines indented 0.5 inches / 1.27 cm).

### Alphabetical Order
- The reference list is ordered alphabetically by the first author's surname.

## Output Format

When checking or formatting references, always present results clearly:

**For a single reference:**
```
📝 In-text citation (parenthetical): (Author, Year)
📝 In-text citation (narrative): According to Author (Year)
📖 Reference list entry:
Author, A. A., & Author, B. B. (Year). Title of work. Publisher. https://doi.org/10.xxxx/xxxx

✅ Verified — [or] ⚠️ Could not be verified — [explanation]
```

**For checking a reference list:**
Present each reference with:
- The original version
- Any errors found (with brief explanations)
- The corrected version
- Verification status

**For batch operations (many references):**
Use a structured summary:
- Total references checked
- Number with errors (with a list of corrections)
- Number verified / unverified / partially matched
- The full corrected reference list

## Common Errors to Watch For

When checking references, pay special attention to these frequent mistakes:

- Missing or incorrect DOI format (should be `https://doi.org/...`, not `doi:` or a bare number)
- Incorrect italicisation (book titles should be italic; article titles should not; journal names and volumes should be italic)
- Missing `&` before the last author
- Including the place of publication (no longer needed in APA 7th)
- Using `Retrieved from` before a URL when a publication date is available (only use retrieval dates for undated sources)
- Incorrect capitalisation in titles (should be sentence case for most titles, title case for journal names)
- Missing issue number or volume number for journal articles
- Including database name/URL for articles that have a DOI
- Using `et al.` in the reference list (never do this — always list all authors up to 20)
- Missing square bracket descriptions for non-standard sources (e.g. `[Video]`, `[Doctoral dissertation, University Name]`, `[White paper]`)

## Important Reminders

- **Never fabricate references.** This is the single most important rule. Every reference must be real and verifiable. If asked to generate a citation and you cannot find a real source, say so explicitly.
- **Always verify when asked.** Use web_search and web_fetch to check that references exist. A reference that looks plausible is not the same as a reference that is real.
- **Be precise about formatting.** APA style is exacting. A misplaced comma, a missing italic, or a wrong capitalisation pattern is an error. Point these out kindly but clearly.
- **Explain your corrections.** When fixing errors, briefly explain what was wrong and why the correction follows APA 7th rules. This helps the user learn, not just get a fixed list.
- **Consult the reference file.** For type-specific formatting templates and notes, always read `references/apa7-formatting-rules.md`. Do not rely on general knowledge alone — the details matter.
